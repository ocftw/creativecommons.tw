第七十二期：Data-as-a-Service 角度看 Open Data
第 73 期 2012-04-06 出刊

這是由台灣創用CC計畫出版的電子報，創用CC，是 Creative Commons 的中文名稱，希望透過「保留部分權利」的彈性授權，讓創作得以更加流通。「網誌選輯」選出相關網誌 (Blog、部落格) 的最新文章、「CC 專題」提供創用 CC 專題報導。

本期專文由工業研究院服務與系統科技中心的符立典研發經理撰寫。本文將從Data-as-a-Service（DaaS）的視角，觀看Open Data議題。 作者點出在良好的數位化基礎下，資料的再利用性及其取得管道的便利性，是服務提供者提升資料價值的關鍵。 另外，提到DaaS營運的分析思考，得以永續的重點，就是必需取得財務平衡。 最後，帶出軟體產業對DaaS和Open Data的期望，並將近期推動的AccessTaiwan計劃經驗與讀者分享。

CC專題：Data-as-a-Service 角度看 Open Data


符立典/文

Data-as-a-Service

提供資料的服務並不是新鮮事，圖書館提供的借閱服務正是其中的典型代表。專業年鑑、產業分析報告、甚至新聞、報章雜誌等傳媒也是都是以各種不同型態存在的 Data-as-a-Service (DaaS)」。這類服務的特性是：以資料或資訊為產品、以便利的資訊管道為服務手段。

DaaS的價值在於：

1. 資料的真確性(Integrity)：高品質的資料必須同時具備「真實、正確、完整、一致」四種特性。例如新聞報導的內容必須是真實的，而且不應該斷章取意、或是前後不一。
2. 資料的珍貴性：珍貴或不易取得的資料，自然具有其特殊價值。
3. 資料的再利用性：從人類的知識史來看，資料被再次利用所創造的價值遠大於資料的存在價值。那麼，愈容易再次被重新利用的資料，便合理的具有較高的價值。
4. 取得管道的便利性：資訊的價值自然與管道的便利性、即時性有關。

承上，若服務提供者想要提升資料的價值，則應考量在現今良好數位化基礎的前提下，資料的再利用性及其取得管道的便利性，理應是這四個維度中較容易提高的兩個項目。的確，我們也可以發現世界各國的Open data推動者中，對於「開發者」這種使用者族群幾乎皆採用REST + JSON的套裝組合來提供資料服務。

REST (Representational state transfer) 原則上採用HTTP或是SOAP RPC (Remote Procedure Call )作為其通訊協定，故幾乎所有具網路連線能力的設備都可以存取。 JSON (JavaScript Object Notation) 則是一種極簡單的資料表達手段，適用於數位資料的序列化及反序列化(Serialization & Deserialization)。目前已有超過45種程式語言的API支援，當然包含主流的JAVA、C#、Object C、PHP、Python等常見的程式語言。 DaaS提供者應該考量採用這樣的模式提供資料給客戶或使用者。因為這模式至少是目前對於資料使用者而言，最便利的管道；也是最容易再被開發二次利用的格式。  

DaaS 營運模式

不論是實體DaaS或是數位DaaS，其營運模式皆包括使用者免付費及使用者付費兩種模式，而其下又可再細分不同的營運模式。

1. 使用者免付費模式：

a. 政府支付營運費用或公益團體贊助：例如政府開放資料(open data)、圖書館的基本借閱服務或公共電視。這類型的DaaS基於社會福利。所以由政府或由公益團體贊助其營運成本。
b. 廣告收入支付營運費用：例如爽報。

2. 使用者付費模式：

a. 依使用量收費：例如訂閱某一個時間區段的期刊、雜雜；或是依章節目錄購買的產業分析報告。API-based的DaaS也可以採用Pay-per-call的模式依照對API服務的呼叫次數做為收費標準。
b. 依資料型態或類型收費：具有較高再利用性的資料收取較高的費用，而具較低再利用性者則可以免費提供。例如，比較PDF格式與JSON格式，JSON應較易被再次利用，故具有較高的價值。
c. 依時效性收費：時效性代表的是資料的珍貴性，而且即時資料和過時資料在不同的場合下其價值也會有不同的判定標準。例如，即時的股市報價資訊的價值非常高，10年前的股價資訊大多可以免費取得。相反的，即時新聞取得的難度要比10年前的特定新聞低，所以10年前的「舊聞」反而可能需收費才能取得。

不論採用哪一種營運模式，DaaS都必需要取得一種能至少讓財務平衡的資態，才得以永續經營。

Open Data - 一種完全開放的 DaaS

政府開放資料(open data)近年來在歐美盛行，但是對於相對較保守的華人文化而言，要完全仿效或是快速推動應該會產生很大的阻力。不論現階段政府對於open data的準備度多高，近期看見的頻頻動作都是個值得期待的開始。政府或法人的角色，應在於提供一個良好的環境，讓open data類型的平台或基於open data的應用得以培育發生，並產生可自給自足營運的可能性。 現階段，台灣中央或地方政府大多數(除了台北市及內政部)對open data都沒有發表相關方針。在公部門手中所擁有的資料有很大比例可能尚未正規化，或甚至尚未數位化。在公部門全面數位化的進行政府open data過程中，法人或民間團體可以扮演推手與技術協助的角色。

而AccessTaiwan是近期一個工研院內部團隊自行發起的計劃，其目的是透過DaaS的手法把台灣的文化、美景和觀光的資訊推向全世界。期望世界各地的 App 開發者可以取得這些難以取得的內容後，自由地發展出多樣的應用。可能是為國外背包客設計的台灣大眾輸輪轉乘工具、LBS (Location-Based Service, 基於地理位置的服務)遊戲、民宿優惠收集、甚至是台灣文化教有的小工具。這些潛在應用的多元性，實維繫在資料來源有多「open」。

軟體產業對DaaS和Open Data的期望

軟體本身與其內容的維護是應用服務產生價值的兩個來源。在軟體創業團隊有限的資源下，應將人力緊縮專注於軟體本身及服務流程的發展。若能從外部取得高品質、易整合的內容，對軟體創業團隊具有絕對性的綜效與價值。

在缺乏這類型的DaaS提供者的目前，政府可以將公部門開放資料做為一個重要的指標性資料提供平台。這個平台的價值不只在於將資料還給社會、或是公開各項應公告的施政資訊；更大的價值在於，這提供了台灣軟體產業一個前所未有的培育環境。除了政府應具前瞻性的思維與環境建構者的使命外，民間團體和法人的協助也是不可或缺的助力。如此一來，在軟體產業鏈的分工與上下游關係亦將更加明確與專業化。如同台灣過去在電子業可以形成完整的產業鏈及產業聚落，軟體亦可以成為台灣下一個被世界注目的焦點。
